What on earth is flexbox?

  • display: flex;

Flexbox is a one dimensional layout property unlike grid for laying out items in rows or columns. It helps layout and distribute space between elements dynamically for unknown sizes, gence the term flex. In order to set a container or section as flexbox, you will need to change the display property value to flex as shown above.

Flex Direction

  • flex-direction: ;
    • row
    • row-reverse
  • flex-direction: ;
    • column
    • column-reverse

When it comes to flexbox its important to understand the terminology Main Axis & Cross Axis. The main axis can be either a row or a column. The main axis can go from left to right or right to left as well as from up to down or down to up. All of this is controlled with with the property flex-direction and the value being either row, row-reverse or column, column-reverse.

Now we head over to the cross axis which is opposite of whatever the main axis is. To elaborate on this, if you main axis is set to row your cross axis will be column. In simpler terms if your main axis goes from left to right or right to left then your cross axis will be up to down or down to up.

However the flex-direction property above has 4 values, row & row-reverse or column & column-reverse, that control the direction of the main axis. So row would go from left to right and row-reverse will go from right to left, while column will go from top to bottom and column reverse will go from bottom to top.

Justify-Content

  • justify-content: ;
    • flex-start
    • flex-end
    • center
    • space-between
    • space-around
    • space-evenly
    • baseline

Justify-content is what controls the positioning of content on the main axis. If the main axis is a row, it will control the positioning of content from left to right or vice versa. Whereas if the main axis is a column it will control the positioning of content from top to bottom or vice versa.

There are a few values that that can be used with the justify-content property such as, flex-start which position the content at the beginning of the main axis. Though depending on your flex-direction, this can be either left to right or right to left and same goes for top to bottom or bottom to top. The most important is to know that flex-start will position content at the beginnig or start of the main axis.

Flex-end like flex-start, just positions the content at the end of the main axis.

Center positions the content on the main axis on its center.

Space-between separates content to either maximum end therefor creating a space in-between them.

Space-around provides a space on either side of the element, meaning left and right. This can lead to certain elements looking uneven.

Space-evenly like space-around provides space on either side of the element, however it keeps a consistant even spacing. This resulting in a even look and feel.

Baseline positions multiple elements according to the baseline of text or paragragh elements.

Flex-Wrap

  • flex-wrap: ;
    • wrap
    • wrap-reverse
    • nowrap

Flex-wrap like flex-direction which we know controls the direction of the main axis, this however controls the cross axis direction. It also controls whether or not elements should wrap. When elements wrap on multiple rows or columns, you can change the direction with either wrap, which depending on the initial main axis direction will be either top to bottom, or left to right. Utimately wrap-reverse will reverse the result of the wrap. You can also set a flexbox or element specifically to nowrap to stop any wrapping from occuring.

Align-items

  • align-items: ;
    • flex-start
    • flex-end
    • center
    • baseline

Align-items does the same thing as justify-content however is positions content on the cross axis. It also doesn't have as many values as justify-content as seen above.

Align-Content & Align-Self

This brings us to align-content & align-self. Both these properties still work on the cross axis and manages the space inbetween rows and columns. While align-self is a property given to individual or independantly to elements allowing them to position themself separate from the flexbox or container.

Flex-Basis, Grow & Shrink

  • flex: grow / shrink / basis;
    • flex-grow: ;
    • flex-shrink: ;
    • flex-basis: ;

Flex-basis, grow or shrink are all properties that are applied to individual or independant elements.Flex-basis defines the initial size of an element, much like width or height, flex-basis controls both of these depending on the flex-direction. If your main axis is a row, then flex-basis controls the width of the element and so for column it controls the height.

Though why not use width or height instead of flex basis? Well, the answer lies in when you scale you browser down past the initial size. With flex-basis it will continue to shrink, while having a width or minimum width or height will force your browser to scroll left & right or up & downto before the content.

Flex-grow allows elements to eat up any remaining space. Flex-grow accepts a unit-less value.

Flex-shrink shrinks specified elements faster than others. Like flex-grow it accepts a unit-less value.

Flex Shorthand

This brings us to the flex shorthand, because these properties are commonly written together, there is a shorthand named flex which accepts 3 inputs, starting from grow, shrink and basis.

Responsive design & media queries intro

  • @media (min-width: #px) { ... }
    • @media (min-width: 576px) { ... }
    • @media (min-width: 768px) { ... }
    • @media (min-width: 992px) { ... }
    • @media (min-width: 1200px) { ... }
  • @media (max-width: #px) { ... }
    • @media (max-width: 1199.98px) { ... }
    • @media (max-width: 991.98px) { ... }
    • @media (max-width: 767.98px) { ... }
    • @media (max-width: 575.98px) { ... }

Responsive design refers to being able to create one website with one style sheet for all screen sizes. It should be responsive towards different viewing platforms without needing to make a completely separate website.

The power of media queries

This brings us to media queries. This is how we control the breakpoints for where we want the website to change depending on screen size or device type. Most commonly screen width, device type or device orientation. The query syntax starts with @media as seen above.
