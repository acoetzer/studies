What exactly is html5?

With HTML5 one would think that it is a different, standalone version of HTML, however it is not, it is however a form or evolution of HTML. The term represents two different concepts, the fact that its a new version of the HTML language. New elements, attributes and behavious but it also introduces new technologies that allow for building better and larger web sites and applications.

The wider and new technologies would include things like new stuff for css, or 3d graphics, upgrades for performance and enhancements.

Block vs Inline elements - Div & Spans

Before discussing <div> and <span> elements, we need to look at 2 keywords, inline & and block elements. These keywords determine how a given element fits into the flow of the document. An inline element are elements that can co-exist on the same line, something like in the example of 2 anchor tags. Whereas the block element will want its on space, something like a paragraph or heading.

This brings us to the 2 elements <div> and <span>. Both elements have a opening and closing tag. The <div> element acts like a generic container, I say generic as we will see a little later that there are other containers known as semantic elements. So the <div> element acts like a block level element, much like the paragraph element, to whatever content is inside its tags. <div> stands for the content division element.

This brings us to the <span> element. The <span> is another generic container but it is what is known as an inline level element. You would wrap this around any sort of text or content that we want to single out later on to style with css.

Odd assortment of elements - hr, br, sup & sub

Just some assortment of elements that you would not really use regularly.

hr element

The <hr> or horizontal rule or thematic break, places a horizontal line across the document to indicate separation between 2 sections, usually you would style this with css by adding a border at the bottom instead of using this element.

br element

The <br> or line break element is use to break lines of text, again not something you should be using as paragraphs already act this way because its a block level element.

sup element

The <sup> or Superscript element acts like the exponent operator or behaviour. it allows for trademark symbols® to be entered in correctly.

sub element

The <sub> or Subscript pushes its content down.

Entity codes

Entity codes or HTML Entities are ways for us to display certain characters that are already reserved for html. So for me to write a greater than or less than sign without HTML thinking I want to add an element, I would need to use the ampersand symbol followed by whatever code and a semi-colon. For example & lt ; (just remove the spaces).

Intro into semantic markup

The word Semantic means "relating to meaning", so semantic markup is meaningful markup or markup that relates to the meaning of the content. When we look at <div>, this has no meaning, hence why we called it generic, whereas something like <section> or <article> has meaning for whatever content is around it.

The reason why it's better to do it this way, rather than just a bunch of divs, is that it helps not only for screen readers or finding blocks of code, but seo as well, helping search results.

A few basic semantic markups to take note of is:

  • <header>
  • <nav>
  • <footer>
  • <main>
  • <section>
  • <article>
  • <aside>
  • <details>
Although they have different names, they don't really do anything different from a div or span apart from just being semantic to its contents. Some semantic elements have some rules attached to them, for example article needs a nested header.
