Intro to For Loops

  • for (initialExpression; condition; incrementalExpression) {
    }
  • for (let i = 0; i <= 10; i++) {
    • console.log(`i is ${i}`)
    }

The whole point of JavaScript Loops is to repeat code. So instead of writing or repeating code 10x, Loops allows us to automate it and repeat the same code, or you can sum numbers together, or iterate over an array. There are multiple loop types in JavaScript. for loop, while loop, for...of loop and for...in loop.

The for loop has a syntax structure as follows: 1st is the initialExpression closed by a semi-colon and then the2 nd portion is the condition, which again is closed by a semi-colon and followed by the 3 rd portion incrementalExpression. Although the syntax says incremental expression, you can decrement or divide or multiply.

The above example displays a for that will print the numbers from 0 to 10, and once it is equal to 10, it will stop.

More For Loops Examples

  • Even number
  • for (let i = 0; i <= 20; i +=2) {
    console.log(i)
    }
  • Odd number
  • for (let i = 1; i <= 20; i +=2) {
    console.log(i)
    }
  • Counting backwards or deducting
  • for (let i = 100; i >= 0; i -= 10) {
    console.log(i)
    }
  • Multiplication
  • for (let i = 1; i <= 1000; i *=10) {
    console.log(i)
    }

In the above examples, we see a simple for loop where you can print either even or odd numbers simply by adding 2 in the incremental value, and manipulating i for the odd value. There are other for loops such as deducting a value and mulitplying. Its not strictly for addition, although thats most common.

We can write this together with the if statements inside a for loop as so:

  • Using if statements
  • for (let i = 0; i <= 10; i++) {
    if (i % 2 === 0) {
    console.log(`${i} is Even`)
    } else {
    console.log(`${i} is Odd`)
    }
    }

The Perils Of Infinite Loops

  • for (let i = 20; i >= 0; i++) {
    console.log(i);
    }

There are times when you encounter an infinite loop. Its a loop that does not stop. The above looks at how one could encounter something like an infinite loop. It's not the end of the world however it can cause your browser to lock up.

Looping Over Arrays

  • Iterating over an array
  • const days = ['Monday', 'Tuesday', 'Wednesday', 'Thursday', 'Friday', 'Saturday', 'Sunday']
  • for (let i = 0; i < days.length; i++) {
    console.log(i, days[i]);
    }
  • Iterating over an array backwards
  • const days = ['Monday', 'Tuesday', 'Wednesday', 'Thursday', 'Friday', 'Saturday', 'Sunday']
  • for (let i = day.length - 1; i >= 0; i--) {
    console.log(i, days[i]);
    }

Most often in JavaScript we loop over an array. Also known as Iterating over an array. When iterating over an array, we usually don't know how many elements are within the array. So the way we get the value of items within the array is by using the length property. We begin at 0 as when indexing an array it starts at 0.

Looking at the examples above we know that we can determine the length of an array using its property length and giving the index a value of 0. Though we can also go backwards in reverse, by giving the index value the array length minus 1, as length counts from 1 and not 0.

Nested Loops

  • Basic nested for loop
  • for (let i = 0; i <= 3; i++) {
    console.log(i);
    for (let j = 1; j <= 5; j++) {
    console.log(j)
    }
    }
  • Nested loop iteratting over a Nested array
  • const people = [
    ['Vanya', 'Andre', 'Dary'],
    ['Buddy', 'Diesel', 'Goofy'],
    ['Mom', 'Dad']
    ]
  • for (let i = 0; i < people.length; i++) {
    const row = people[i];
    console.log(i);
    for (let j = 0; j < row.length; j++) {
    console.log(row[j]);
    }
    }

The above is an example of a nested loop. The idea of a nested loop, is essentially a for loop inside a for loop, or a for loop inside a while loop. Though looking at the first example of a basic nested for loop. The idea is that with a single iteration you have multiple inner iterations. In this case The outer itaration will happen 3 times, while for every outer iteration, there will be 5 inner iterations.

Another thing to note is, we use i j and k, where i stands for index, though it can be anything you want it to be, for example inner, outter. j and k are usually also used as its the next letter up from i.

We use nested loops, when wanted to iterate over nested objects, such as a nested array. The second example, showcases how a nested for loop is used to iterate over the nested array. We first iterate over the first array using the i, therefor i captures the initial index of the array. We then save that to another variable inside the for loop known as row.

We then iterate over row using the j variable. Essentially row = people[i], we dont need to save it to a variable, as within the last for loop we can specify inner indices as so people[i][j]. The process essentially repeats itself for the nested loop.

Another Loop: The While Loop

  • while (condition) {
    console.log('something');
    condition++;
    }
  • const num = 1;
  • while (num <= 5) {
    console.log(num);
    num++; }
  • Same as
  • for (let i = 1; i <= 5; i++) {
    console.log(i);
    }

While loops are much simpler than for loops, all you have is the while statement and any condition. So while (true) do something. This will repeat code until the loop either breaks or ends by evaluating to false.

When to use a while loop rather than a for loop? That is when the outcome is truly variable. So as an example, if you playing a game of chess, it will switch between player 1 and player 2 until one of them wins. When is that?, that could be in a few moves or it can take a long time. Thats when something is truly variable.

Whereas a for loop is when you know how many iterations it will take. The while loop and for loop above are exactly the same, and although we dont use any of these loops like this, the for loop is more common for this as we know how many iterations it will take.

An interesting thing to note about while loops, or any loops in general, placement of things for example console.log() and num++. If for example console.log() is above num++, it will start printing from 0 whereas if it was after, it will start print from 1.

The Break Keyword

  • let userInput = prompt('Say something');
  • while (true) {
    if (userInput === 'stop') {
    break;
    } else {
    userInput = prompt(userInput);
    }
    }

There are times when you will want to break out of a loop, and the keyword for breaking out of a loop is break;. This just beraks out of a loop, as the example above if any condition is met, then it will break out of the loop. Otherwise it will just copy the users input and present it back to them.

The Lovely For...Of Loop

  • for ... of loop
  • const classroom = ['Andre', 'Vanya', 'Dary', 'Colt', 'Keele'];
  • for (let students of classroom) {
    console.log(students);
    }
  • for loop
  • for (let i = 0; i < classroom.length; i++) {
    console.log(classroom[i])
    }
  • nested for ... of loop
  • const classroom = [
    ['Andre', 'Vanya', 'Dary'],
    ['Mom', 'Dad'],
    ['Buddy', 'Diesel', 'Goofy']
    ]
  • for (let row of classroom) {
    for (let student of row) {
    console.log(student)
    } }
  • Nested for loop
  • for (let i = 0; i < classroom.length; i++) {
    const row = classroom[i];
    console.log(i);
    for (let j = 0; j < row.length; j++) {
    console.log(` ${row[j]}`);
    }
    }
  • Iterate over Iterable things
  • for (char of 'hahaha') {
    console.log(char)
    }

The for... ofloop is a newer concept in JavaScript, and is not supported with certain browsers. The for of loop, is a nice and easy way to iterate over arrays and or anything iterable. You would think you can iterate over a object literal, but that is not iterable with the for of loop.

The syntax of a for of loop is, for (let varaible of iterable) {statement}. When should you use a for loop instead of a for of loop. That is when you need the index value of something.
