Introducing Arrays

  • const array = [];
  • const days = ['Monday', 'Tuesday', 'Wednesday', 'Thursday', 'Friday'];
  • const mix = ['Weekend', true, 65, 0.99999, false];

Our very first data structure, JavaScript Arrays. A data structures is simply a collection of data, can be any data type. An array is an order collection of values. Arrays are enclosed with [Square Brackets] and you seperate each value by a comma.

You can assign a variable an empty array. Arrays also have an index value, unlike the index of a string where it returns a single character. This returns an entire value at whatever index was specified, basically an entire value. You can store multiple different and seperate data types within a Array.

In short, the 2 key concepts of arrays is that Its a collection a values and second its an ordered collection of values.

Array Random Access

  • const days = ['Monday', 'Tuesday', 'Wednesday'];
    • days[0] Monday
    • days[0][0] M
    • days.length 3
  • const colors = ['Rad', 'Blue', 'Orange'];
    • colors[0] = 'Red'; ['Red', 'Blue', 'Orange']

We know that arrays are indexed just like strings, however it does not return just a single letter but the whole value inside the array. We can further index or chain onthe array to get the result of a single character as seen above. string indices returns characters, and array indices returns values. This is where the order fits in.

When trying to index a number greater than the amount of elements in the array you get the result undefined. We know undefined is javascripts way of saying it doesn't know.

You can use the indices to change the value stored inside an array. Its important to note, that you cannot do this with strings, string are not mutable, even within a let variable. However with arrays you can change the values using indices.

quick note, you can index a greater value than the amount of elements inside an array and add a value to it. JavaScript will add that value to the end. As an example, if you index colors above, colors[10] = 'Purple';. The result will be ['Red', 'Blue', 'Orange', empty x 7, 'Purple']. Notice the 7 empty slots added to the array which are all undefined when indexed.

Push & Pop

  • arrayVar.push('add something at end');
  • arrayVar.pop(' remove something at end');
  • arrayVar.shift(' remove something at start');
  • arrayVar.unshift('add something at start');

Coming to array methods, we saw how to create an array, change elements within an array using indecises, however there is an easier way to add or remove something to the end or start of an array without guessing the index.

To add something to the end, we use a method called .push('ARG'), this adds a value automatically to the end of an array.

To remove something from the end of an array, we use a method known as .pop(), Pop by default will remove the last value in the array without using an ARG.

Shift & Unshift

Like the above methods this is used for the beginning of an array. To add something to the beginning of an array, we use .unshift('ARG').

To remove an element from the front, we use a method known as .shift(). Again, no ARGs needed.

Looking at further methods for arrays, You can .concat(array#2) 2 arrays together. You can just like variables, do the .indexOf('element'), this giving you back the index of a specified element.

Another method can give a true or false value, as .includes('element'), or you can reverse an array with the method .reverse().

Slice & Splice

  • arrayVar.slice(indexStart, [indexEnd]);
  • arrayVar.splice(indexStart, deleteCount, addItem);
  • arrayVar.sort();

.slice() & .splice(). Slice is a way for you to get a portion of the array, you can provide a start index, but you do not have provide an end index. firstly, if you don't provide an end index, it will give you everything from the starting index all the way to the end of the array. You also provide the .slice(-3) a negative value and it will give you in this case, the last 3 values of the array.

.splice() on the other hand is a way for you to add elements inbetween other elements inside an array.

.sort() sorts an array, though the array sort method is at this moment useless as the default way it sorts elements is by converting the elements to strings and then sorting it by its UTF-16 code unit values. As a quick example [1, 30, 4, 21, 10000].

Reference Types & Equality Testing

  • 'hi' === 'hi' true
  • ['hi', 'bye'] === ['hi', 'bye'] false
    • [1] === [1] false
    • [1] == [1] false
    • [] == [] false
  • const arrayOne = [1, 2, 3, 4];
    • const arrayTwo = arrayOne;
    • arrayOne === arrayTwo true

Looking at the example above, 2 strings can in memory match each other, however when it comes to arrays, no 2 array is the same.Even though the contents are the same, in the stored memory it is not.

You can think of this like a reference to an array. You can test this by creating another variable with the first array variable inside and it will be the same, as shown above.
