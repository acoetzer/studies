Javascript only has one number data type, unlike other programming languages where they have two data types such as int(integer) for whole numbers and float for decimal numbers. In short, javascript only has 1 type and that includes positive, negative, whole and decimal numbers.

The reason for ints and floats is that it takes a lot more memory to store those numbers especially if its a decimal number. numbers do get a certain amount of space in memory in javascript, something like 1.999999999999999 will eventually become 2, they are not permitted to continue or to take up an infinite amount of space. there is a hard limit in other words.

along with the number data type, you have some common math operators that are built into javascript. Looking at the example above, you see things like Addition, subtraction, division all the way to modulos which is mainly used to determine if a number is odd or even, as an example 89089 % 2 //1 and exponents which are represented with ** two astrix symbols. There is however order of operation, span.highlight{PEMDAS} which stands for Parathesis, Exponents, Multiplication, Division, Addition and Subtraction. So something like 3 + 1 * 9 //12, the 1 * 9 which is the multiplication comes before addition. You can change that by placing a parenthesis as so (3 + 1) * 9 //36, so now the numbers within the parenthesis come first.
