Number

  • Math Operators
    • //Addition
      • 50 + 5 //55
    • //Subtraction
      • 90 - 1 //89
    • //Multiplication
      • 11111 * 7 //77777
    • //Division
      • 400 / 5 //16
    • //Modulo
      • 27 % 2 //1
    • //Exponents
      • 2 ** 4 //8

Javascript only has one number data type, unlike other programming languages where they have two data types such as int(integer) for whole numbers and float for decimal numbers. In short, javascript only has 1 type and that includes positive, negative, whole and decimal numbers.

The reason for ints and floats is that it takes a lot more memory to store those numbers especially if its a decimal number as decimal numbers are considered much larger. Though, in JavaScript numbers do get a certain amount of space in memory, so something like 1.999999999999999 will eventually become 2. They are not permitted to continue or to take up an infinite amount of space. there is a hard limit in other words.

With the number data type, you have some common math operators that are built into javascript. Looking at the example board above, you see things like addition, subtraction, division all the way to modulos which is mainly used to determine if a number is odd or even, as an example 89089 % 2 //1 (which is considered odd //0 is considered even) and exponents which are represented with (**) two astrix symbols.

There is however order of operation PEMDAS which stands for Parathesis, Exponents, Multiplication, Division, Addition and Subtraction. So something like 3 + 1 * 9 //12, the 1 * 9 which is the multiplication comes before addition. You can change that by placing a parenthesis as so (3 + 1) * 9 //36, so now the numbers within the parenthesis come first.

  • Not a Number
    • 0/0 = NaN
    • 1 + NaN = NaN
    • NaN + NaN = NaN
  • typeof

NaN is a concept in JavaScript that represents a value that is Not a Number but at the same time is considered a numeric value or in other words a number. We can check this with the use of another operator known as typeof followed by a single operand.

There will be times when you doing something and it will output NaN and this could be due to a value being 0 and you not knowing about it. There could times where user input is with characters and you trying to do math with letters and you receive the results of NaN.

Variable & Let

  • To define a variable:
    • let someName = value;
  • To update a variable:
    • someName = newValue;

What is a variable? A variable is like a container that holds a value which is given to it. An example would be let numChickens = 4;, this is telling JavaScript that you are, first of all, defining a variable and giving that variable a name with a value. So that, whenever you call upon the variable, it will out put whatever value was given to it.

It is important to note the semi-colon ; at the end in JavaScript, In the terminal you don't really need to use it, but when writing a js file, its important to have.

You can also update the variable someName = someName + newValue, its important to note a distinction here, and that is defining a variable and updating a variable. You will notice when defining a variable, you use something like let. This is telling Javascript of a new variable, hence defining. However, you do not need to define the variable again just to update it, that is because the variable is already there and you just need to refer to it.

You can do math with variables numHens 5 + numRoosters 1, refer to video if this doesn't make sense, however each variable has a number value, which would equate to 6 in this instance. You can also, as an example say, numHens + 1, you can add a variable and a number together, considering that the variable is a number, otherwise you will end up with a NaN.

Updating Variables

  • let score = 0;
    • score = 5;
    • score = score + 5;
      • score += 5;
      • score -= 5;
      • score *= 5;
      • score /= 5;
  • Incrementor
    • Postfix Incrementor
      score++
      Prefix Incrementor
      ++score
  • Decrementor
    • Postfix Decrementor
      score--
      Prefix Decrementor
      --score

When it comes to updating variables and more specifically number variables, and working from the above example board, if score was set to 0 and we wanted to add 5 to the score, we can change or update the variable as so score = 5;. However you can continue to manually change it. The next possible way to update the varibale is by giving the variable an addition as so score = score +5;, so now whenever something happens it will update the score by 5.

Another way to do this is by writing a shorter syntax, score += 5;, this is the same as the previous example, just much shorter. You can do this for other operators as well.

However, there is another way, though this only increments or decrements the variable by 1 which is quiet common and that is using what is known as an pre/postfix incrementor or decrementor. An example of this would be score++ or ++score the same as score-- or --score.

Booleans

  • let isLoggedIn = true;
  • let isLoggedIn = false;

Booleans are fairly straight forward, a boolean is another primitive type, that stores a true or false value. Note that they are written with lower case letters, while in other programming languages, they are written with the first letter as capitalized.

Changing Variable Type

  • let someName = true;
  • let someName = 3452

In JavaScript, variables can change its type from a boolean to a string or numner, unlike other programming languages, where if a variable is set as a boolean, it must remain as a boolean. JavaScript, does have something like TypeScript that enforces these rules, but plain old JavaScript does not.

Variable Naming & Conventions

  • Hard Rules
    • No Spaces within an Identifier
    • Dont Start with a number with an Identifier
  • Best Practices
    • User Camel or Snake Casing
    • Meaningful Identifiers

When it comes to Identifiers or naming your variables, it is important to note some hard rules whereby that you cannot start an identifier with a number, nor can you have spaces within your identifier. So something like let some name = false; is not allowed, nor is let 1user = true; is not allowed or permitted.

However, let user1 = value; or let username = value; is considered fine. Though, there are some best pratices or convention rules to use when writing identifiers, and those are things like camel or snake casing like so let userProfileStatus = value; or let user_age = value;. Another good practise to have some sort of meaning in your identifiers and not just 1 letter characters or something like this, let isLoggedIn = true;, where you know when you use is in the identifier, it will relate to a boolean type.

Also its best to have meaningful variable names, but also not overly long identifiers.
