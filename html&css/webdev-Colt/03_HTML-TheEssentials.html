Introduction to HTML

Taking a look at this saying, the Purple Dino Danced. We can break down what HTML is compared to and how to explain what HTML is. HTML is the 'what'in this sentence, in otherwords the dino, whereas the purple is css and danced is js. HTML is what is being displayed or viewed, so the in short its the what.

HTML is known as a markup language and stands for Hypertext Markup Language. It is not a full blown programming language with complex logic, It's a way to mark up a document (plain text) and have a way to indicate what is bold, italic or what is a link as an example, through what is known as elements or HTML elements.

To write HTML we use what is know as HTML elements, as exmaple:

  • <p>
    • This represents the paragraph element
  • <h1>
    • This represents the main header on the page
  • <img>
    • This embeds an image
  • <form>
    • This represents a form
These are some of the elements used to write HTML. Although above doesn't illustrate this, but we generally wrap some form of content with what is knowns as <opening tag> and a </closing tag>.

An example would be something like this <h1>This is a header</h1> to represent the start and end of the main header. You can also nest tags within elements, using the same example as before, you can make the header bold, like so <h1><b>This is a header in bold</b></h1>.

Paragraph Elements

The first element we look at is the <p> for paragraph element, and this doesn't mean its only for text purposes, you can have a paragraph for things you wish to group together. That could be a paragraph of images or different pieces of a form. However for now we are only focused on textual paragraphs.

The paragraph element has a opening and closing tag to indicate the start and end of said paragraph or group. A paragraph can also contain nested elements such as bold, italics and various other elements in HTML. The paragraph element is a block level element, that take take up there on block.

Heading Elements

A heading element is a way to provide headings on documents. There are some rules however, each document can only contain 1 <h1> heading, as well as a specific structure to follow.

The structure is as follows:

  • <h1>H1 Heading</h1>
    • <h2>H2 Heading</h2>
      • <h3>H3 Heading</h3>
        • <h4>H4 Heading</h4>
      • <h3>H3 Heading</h3>
        • <h4>H4 Heading</h4>
The structure above follows a form of hierarchy.

You can have multiple h2-h6 headings as illustrated above, though the nested headings should follow the heirarchy form. So never use a <h1>-<h6> for only the size, as these headings have a default size and can be changed using css.

Just like the paragraph takes up its own space due to it being a block level element, so is the headings element. Anything that comes after the closing tag will be pushed to its own line.

HTML Boilerplate

HTML Boilerplate or otherwise known as the HTML Skeleton, is essential to every .html document. The boilerplate follows a structure of a Doctype, html element as well as a head and a body. Here is an example of the essential skeleton needed.

  • <!DOCTYPE html>
  • <html>
    • <head>
      • <title></title>
    • </head>
    • <body>
    • </body>
  • </html>

Starting from the top down,the <!DOCTYPE> is something of a flag to your browser indicating that this page is using HTML5. This does not need a opening and closing tag. The <html> element or otherwise known as the root element, is the top level element on the document and every other element is a decendant of this element. However there is a rule, you are only allowed 1 head element and one body element.

That brings us to the <head> element. The head element contains things that wont show up on the page, such meta data, information about the web page like <title> and various other things like loading in css or scripts. Just below the head we have the <body> which represents all of the content in out document and it can include any of the elements we have learn't so far.

List Elements

There are 2 kinds of list elements, a <ul></ul> un-ordered list or bulleted list and a <ol></ol> ordered list or numbered list. These list elements contain a child element <li></li>. You can also nested <script> or <template>.

You can also nest another list within a list. You can also mix up a ordered list with a un-oreder list.

Anchor Tags

The anchor tag is by far the most crucial element in HTML, as it is used to provide links to other pages, or to jump to a specific heading on a page. The <a></a> tage is opening and closing but also contains something known as a attribute. An attribute as seen here <ahref="www.google.com">google</a> tells the anchor where to go.

You will notice the attribute sits inside the tag, while the text being displayed is inbetween the tag. This particular attribute named href, which stands for hypertext reference, will be on every single anchor you make. With anchor tags, you can link to local documents, or a web page.

Images

The <img> element is the first element that does not have an opening or closing tag, however does have attributes to indicatate the source for where to fetch the image being embedded. An example of this would be <img src="/some/location"> or <img src="https://some.co/location">

The img element can hold further attributes to size the image or alt text as so <img src="/some/location" width="200px"> and <img src="/some/location" alt="Alt text for screen readers">.
