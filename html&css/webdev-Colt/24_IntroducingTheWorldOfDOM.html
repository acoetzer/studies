Introducing the DOM

  • DOM
    • Document Object Model
  • console.dir(document);

The Dom stands for the Document Object model. This is JavaScripts window into the contents of a webpage. It's basically a bunch of Objects you interact with via JavaScript.

The Document Object

  • Imagine an image here that has a data tree of objects in the form of html elements

The way the Document Object works, is when our browser loads, part of that process is taking the html elements and style rules and converting that to JavaScript Objects which is found in the document object.

Above we said to imagine a data tree where at the very top of this tree lies the document object and next to it the HTML markup. When writing the markup, we usually have a parent, child & sibling element such as an ul > li inside it.

Throughout the markup we have many of these things parent/child structures. Well its the same from the document object, it copies or replicates the exact same data structor.

To view the objects, properties and methods inside the document object, you can display really any object the following way console.dir(document). This sort of proves to us that the document is actually an object that holds other objects that hold other objects which hold properties and methods. A vicious looking tree.

The Document Object is created automatically for us by our browsers when we load a new page and one thing to remember the DOM is based upon the contents of the page.

To end this portion, the document object is our entry point to the world of DOM. It contains a representation of whats on the page, plus a ton of useful properties and methods.

It's the root of the Tree!

getElementById

  • document.getElementById('id')

Welcome to the world of seclecting objects. .getElementById('id') is one way for us to select objects in the Document Object. We spoke how how the document object contains a bunch of other objects and properties or methods. This tree like Data Structure.

.getElementById is a method that helps us select these objects by their id. If we think of CSS and its selecting methods, its very similar. This is just how we select things using JavaScript.

Of course there are better ways to select objects inside of javascripr, for example querySelector. However, it is important to note that before we can manipulate anything inside of javascript we first need to be able to select it.

The getElementById requires you to input a string argument of the element id. You will notice most things will require a string value, when it comes to the DOM. If it cannot find the id, it will return a null value.

When selecting, whether it be with ById or ByClass ... and so on. It will always return the Object, even though inside the console, it looks like html, you can verify it with console.dir() command.

getElementsByTagName & ClassName

  • document.getElementsByTagName('tag')
  • document.getElementsByClassName('class')

The other two getElement s... getElementsByTagName & getElementsByClassName are grouped together as they both deal more than one element. Hence the attempt to highlight the s in the names.

Apart from working relatively the same, the only difference is that it returns what is known as an HTMLCollection which holds what looks like an array of objects, however you cannot use array methods on it. You can however iterate over it, as it has indices.

Another distinction is that with getElementById, if something is not found, you have are return with a null value, whereas with selectors that return multiple objects in an array form, you are return with an empty array if it cannot find what you try to select.

querySelector & querySelectorAll

  • querySelector('Tag, #id, .class')
  • querySelectorAll('Tag, #id, .class')

The querySelector & querySelectorAll are new ways in JavaScript to select objects, they exactly the same as how you would expect CSS selectors would work. You can go as far as nth-of-type() as well as a[type="text"].

Apart from the HTMLCollection you get back when using the getElementsBy selectors, here you are returned with a NodeList which is also in the form of an array.

innerHTML, textContent & innerText

  • .innerText = 'The inner text'
  • .textContent
  • .innerHTML = '<b> some text </b>'
    • .innerHTML += '<b> some text </b>'

The above are all properties to Elements or Objects in JavaScript. They all have to do with the content inside of an element.

innerText provides back the text between the opening and closing tags og an HTML element, though compared to textContent it is sensitive to what is showing. So you had to set display to hidden or none, then it would not show that, where textContent would.

innerHTML allows us to change HTML inside a given element or object. It will also display everything including the HTML markup.

These methods allow us to change or set an attribute of a element, and the other allows you to retrieve it.

Changing Styles

  • .style."Here is the same as CSS" Just all in camel case.
  • Example between CSS and JavaScript
    • CSS: font-family:;
    • JS: fontFamily = ''
  • h2.style.fontWeight = '700'

So, when changing an elements style through JavaScripts style property, it will list all CSS properties that can manipulated. The only difference is that, its camelCased. Also it kinda saves it as a inline style property which we know is not very good to do as that can slow down your site.

What we will see later on is that it will be better for JavaScript to change the class of an element and to store the styles in a CSS file under a particular class, and then toggle the classes.

getComputerStyle

  • const h1 = document.querySelector('h1')
  • window.getComputerStyle(h1)

If you wanted to see the font size of a particular element, you can run the command above.

ClassList

  • .classList
    • .classList.add('')
    • .classList.remove('')
    • .classList.toggle('')

Before we mentioned that its better to toggle a class on an object rather than to place an inline style. .classList allows us to do just that. .classList has a bunch of properties and methods for dealing with classes.

.add('className'), adds a class to an element.

.remove('className'), removes a class from an element

.toggle('className'), toggles a class on and off.

Traversing Parent/Child/Sibling

  • .parentElement
  • .children
  • .nextSibling
    • .nextElementSibling
  • .previousSibling
    • .previousElementSibling

The properties above, traverses you across to different elements. .parentElement selects the parent of the current element. While .children show the current elements children elements.

nextSibling on the other hand returns an empty text node, while nextElementSibling provides you with the next sibling elementVice versa for previousSibling and previousElementSibling

Append & AppendChild

  • .createElement
  • .appendChild()
  • .append()
  • .prepend()
  • .insertAdjacentElement()

There are ways to add elements to the page using JavaScript. Thought before we can add an element to the page, we first need to create it using createElement, as an example const img = document.createElement('img') will create an empty span object/Element.

Once we have created the element we can verify its an object with console.log(img).

Now that we have this empty element, we can either add another element to it, considering we create it first, though in this case we created an empty img element. Now we can fill the img element source. img.src = 'https://somethingOfALinkHere.com'

Once we have we have created the element and added the content and have the element shich we want to append to selected, we can now append it to the page using either .append(), .appendChild(), .prepend() or .insertAdjacentElement().

removeChild & remove

  • header.removeChild(h2)
  • h2.remove()

.removeChild() removes a child element, Though you can remove the current element you have selected, as it was done in the past like this h2.parentElement.removeChild(h2).

.remove() remove the element directly, like so h2.remove() there is no need to pass in anything as a parameter.
