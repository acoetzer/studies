Introducing Object Literals

  • Objects
    • Object are collections of properties.
    • Properties are a key-value pair
    • Rather than accessing data using an index, we use custom keys
  • A property is a key: value pair.
    • agekey: 32value,
  • Not Ordered

JavaScript Objects, Our second data structure. An Object literal is another way to store data using what is known as key: value pairs. Object Literals are not ordered unlike Arrays and this is because you create properties which have a key and value assigned to it, inside the object.

Object literals cannot be indexed, anything inside [] if not in a string, is considered a variable. The most important thing to remember is that properties are key: value pairs.

Creating Object Literals

  • const person = {firstName: 'Mick', lastName: 'Jagger'}
  • const person = {
    • firstName: 'Mick',
    • lastName: 'Jagger',
    • age: 78,
    • nationality: 'British',
    • birthYear: '26 July 1943',
    • isAlive: true
    }

The way you create an Object Literal is by using curly braces, unlike Arrays where we used square brackets. Inside the curly braces, we add a key as seen above followed by a : colon and the value.

When it comes to the value, it can be any data type. You will notice there is no real order to the object literal example above.

Accessing Data Out Of Objects

  • const person = {firstName: 'Mick', lastName: 'Jagger'}
    • person{1} DO NOT DO THIS!
    • person[1] undefined, Can't Index! unless there is a key of 1 then it will convert to the string
    • person[firstName] undefined, is a sking for a variable.
    • person['firstName'] "Mick"
    • person['first' + 'Name'] "Mick", Concatenates first.
    • person.firstName"Mick"

When it comes to accessing data within an Object Literal it is not with with curly braces. The only time we use curly braces is when we declaring an Object Literal.

We use [square brckets] just like arrays, however we do not use indices. We either provide a string value like so person['firstName'] or a .property like so person.firstName, which again keys are properties and properties are key: value pairs.

Quick note to remember is that all keys are converted to strings. Except for symbols.

Inside the [], you can input a expression like so person['first' + 'Name'], This will work, as the expression will run first, which is basically concatenating the 2 strings together to form the 'firstName' string.

You can also input a a varaible like so person[firstName] considering that variable is a string with a key identifier. Though, when it comes to number like so person[1999], this will be converted to a string first and then try to match it with the same string inside the object.

Modifying Objects

  • person['firstName'] = Micky;
  • person.firstName = Micky;

Just like an Array, the inside of a object literal can be cahnged. However the container cannot. Changing data inside an object is self explanatory, you select the object and property and assign it a new value.

The same method we use for updating or changing values, can be use to create new values to be added to an object.
