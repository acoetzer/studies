What Are Events?

Well simply put, events respond to user input or actions. Something like the click of a mouse, a hover or mouseenter. Of course these are the names of the Event Listener which will see shortly, but the properties such as onclick can be viewed just by viewing the object. The way you view the object is with console.dir(object).

There are 3 ways to create events. Starting with and should never really be used, is inline events as seen above. It's realy the same principle as styles, never do it.

Onclick Propery

  • The Events Properties
    • button.onclick = function () {
      alert('You clicked me')
      }
  • const h1 = document.querySelector('h1')

    const hello = () => {
    console.log('hello')
    }

    h1.onclick = hello;

The second option and still not the best method is writing them in the JavaScript file, but working directly with with the properties.

Ofcourse we need to select the elements first in JavaScript before applying the event. Again to view these properties, use console.dir(whatever specific element) and when scrolling down you should see a bunch of onproperties.

You can use multiple events on a single element, example would be button.onclick and button.onmouseenter, though you cannot write the same event for the same element down the line. Sometimes we will need to write separate events as the contents suit the situation better and using properties doesn't allow us to do this. Hence the next better way of handling events.

Quick note you can write a function before hand and then execute it with the event. btn.onclick = func

addEventListener

  • The Evnt Listener
    • btn.addEventListener('event name', func){
      your code here
      }

Moving onto the best way and the third and final way, is the addEventListener. This is the swiss army knife of adding events.

We have seen actual properties, like onclick..., though the best way is to write events with the event listener. The reason we would use something like this is sometimes we add more then one event on the same property or element.

Though when using the properties, we are pretty much setting the value to that and if had to write another, we would be overwritting the first value. Almost like in CSS where specificity or order matters.

The addEventListener can accept a predefined function as well, and often we usually write functions where code in events are repeated, or getting too long.

The most important is the event listener options. A example of this would be, btn.addEventListener('eventName, predefined Function, { once: true }'). The { once: true } is the options among many other options. However in this case, it allows for an event to run once, and then it removes that event.

There is something known as removeEventListener, though for now we wont speak about it.

    Keyboard Events & Event Objects

    • Event Objects ((e/ evt/ event))
      • btn.addEventListener('click', function (e/evt/event)) {
        console.log(e/evt/event)
        }
    • Keyboard Events (code, key)
      • window.addEventListener('keydown', function (e)) {
        console.log(e.code)
        console.log(e.key)
        }

    When we talk about Event Objects, Its important to understand that JavaScript automatically creates this event object when using event handlers. To capture the event object, with in the function parameter we can input anything really, but most do e,evt or event inside console.log(e).

    So inside the the object event, you will see a bunch of properties that relate to mouse clicks, or the location of where we click like clienty & clientx, the target, key, code, just to name a few that we will end up using.

    Though in this instance, we are interested in the key & code properties as this relates to the Keyboard Events object.

    code corresponds toto the location of the physical key on the keyboard. Examples of that would be KeyA, KeyW, KeyD, KeyS & ArrowLeft, ArrowUp, ArrowRight and ArrowDown

    key corresponds to the letter. An example of that would be a, w, d, s, Arrowleft, ArrowUp, ArrowRight & ArrowDown.

    The reason why code and key are important is that we can use the event object to restrict certain keys.

    New Version: Form Events & PreventDefault

    • The Submit event
      • form.addEventListener('submit', function (e) {
        console.log(e)
        })
    • .preventDefault();
      • form.addEventListener('submit', function (e) {
        e.preventDefault();
        })
    • form.addEventListener('submit', function (e) {
      e.preventDefault();
      const username = this.elements.username
      })

    When dealing with form events such as submit, sometimes if not most cases we want to stop the Default behavior of the form.

    When we create a form, we get this action attribute which is where the form will be sent to. However when this happens, it usually changes the page and we lose whatever information was on the page. To prevent this behavior we need to capture the event object and then run its method .preventDefault(). This will stop the default behaviour of the form.

    This is important, as if we want a single page workflow like the example below, preventing the default behaviour is a must.

    Another important thing to know is that the form object element, console.dir(form), has a property called elements that house the name of the elements inside it.

    Event Bubbling

    • something.addEventListener('some event', function (e) {
      e.stopPropagation()
      e.stopImmediatePropagation()
      })

    Event bubbling is when you have multiple events within events. You can imagine bubbles floating to the surface. Well, when you have multiple events inside events, all events are triggered starting from the event being envoked.

    To stop this, you can once again capture the event object and run its method called .stopPropagation().

    Event Delegation

    • Event Delegation
      • ul.addEventListener('click', function (e) {
        e.target.nodeName === 'LI' && e.target.remove()
        })

    Event Delegation allows us to have a pattern to handle events efficiently. Instead of adding an event lsitener to each and every similair statement we can add an event listener to a parent element can call an event using the .target on a particular element.
