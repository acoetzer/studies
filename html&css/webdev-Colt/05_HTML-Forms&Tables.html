Introducing HTML tables

  • <table></table>

In short, tables are a why to show tabular data such as rows and columns, as you would expect to see in a table. In the past tables were not just used for tabular data but used to layout a website as well, whereas today you don't really need to do that.

The way you begin a table is with the table element shown above, there are various child elements that go within a table element.

Tables: TR, TD, and TH element

  • <tr></tr>
    • <td></td>
    • <th></th>

The first set of child elements related to the table element is tr, td, th. tr stands for table row, this adding a row to the data table and the child elements which follows fit into the tr element. The first would be the th element, which represents the table heading cells or header cells.

The second nested element for tr would be the td element. The td element stands for table data cell and these are the cells that would be placed on the table row.

Tables: Thead, Tbody and Tfoot Elements

  • <thead></thead>
  • <tbody></tbody>
  • <tfoot></tfoot>

So like semantic elements, tables have there own set of semantic "divs" in a way. It's pretty self explanatory. thead grouping heading elements, tbody grouping the body of the table, essentially everything else and tfoot grouping footer element or what would considered footer elements.

Tables: Colspan & Rowspan

  • <th colspan="2"></th>
  • <td rowspan="2"></td>

colspan & rowspan are attributes placed within the table element tags, as demonstrated above. The basic idea is that you can make one heading or cell span across multiple rows or coloumns.

The Form Element

  • <form action="/tacos"></form>

The form element in itself is just a container holding various input elements or individual form controls such as text inputs, checkboxes and buttons. Though we group these together in order to create a form. There are various different types of forms ranging from search bars, to login forms. The parent form element <form> which is used to contain input or form controls, is what specifies where and how the data is sent or used, by the use of the attribute known as action.

Common input types

  • <input type="text" placeholder="username" name="username" id="username" minlength="3" maxlength="20" required>
  • <input type="password" placeholder="password" name="password" id="password" minlength="8" maxlength="16" required>
  • <input type="email" placeholder="enter email" name="email" id="email" required>
  • <input type="color">
  • <input type="number" name="username" id="username" min="3" max="10" required>

The most common form control is the <input> element, which is extremely versatile. It can range from checkboxes, color pickers all the way to password and plain text inputs. The way that this works, even though it is one element (input), you change one attribute within the element, and that is the type attribute <input type="text, password, email ...">.

As we see above in the list, we can see various attributes but what do they mean?

Explaining attributes

type="?"

The type attribute controls the type or "what" is being used in this input element. It can range from text, password, color, number, email, and many more depending on what is needed.

placeholder="?"

The placeholder attribute is used to display an example of sort or instruction of what a particular form control is looking for. As an example:

name="?"

Check here for explaining on the name attribute. Essentially the name attribute is a sory of like a key pair value system.

id="?"

The id attribute specifies a unique id for an html element. In short, the id helps connect other elements together, or style sheets to a specific header as an example, or even to javascript, allowing it to manipulate certain elements.

As an example, If you click this word it will place you cursor inside this text elemeent here . This is achieved by using the id attribute and linking one thing to another through the use of unique identitfiers.

This should go without saying, but id's should be unique on any given page or document, you cannot have multiple of the id, nor should you start an id with a number.

min="?" minlength="?" or max="?" maxlength="?"

The min and max attribute control the maximum number in a number controller, whereas the minlength and maxlength attributes control the minimum or maximum number of characters in a text controller.

value="?"

There are various ways value affects different elements within html, however in general a value sets the value of an item. A good example would be a range controller, you can set the value to be 75 and everytime that range appears, it will default to 75.

Another example of the value attribute is that you can give value to somehting that has no form of user input. Something like radio buttons, if submitted, you would see name=on. Even though it has a label connected to it, it just doesnt have a value, so therefor you can provide a value to something without user input.

checked

Short explanation, this is an attribute that gives a checkbox controller or radio controller a a default checked box or option. An example would be this radio button already having one option marked:

required

Check here for explaining on the html form validations attribute. However in short the required attribute, forces certain controllers to be filled before excuting any action, such as submitting.

for=""

Although we havent seen this yet, and can be seen here, the for attribute specifies to which element a label element belongs to.

The all important label element

  • <label for=""><label>

Labels are an essential element to forms, even though some devs dont use them as they should. They use the placeholder attribute instead of a label, the placeholder however is not accessable for screen-readers, while labels are. This could also potentially affect SEO with google search engines.

HTML Buttons

  • <button></button>
  • <button type="button"></button>
  • <button type="submit"></button>
  • <input type="button">
  • <input type="submit">

A button is pretty important to your typical web page, whether they in a form or not. Whatever you put inside the opening and closing tags of a button element, will be used to label the button. Button elements can be placed inside forms however their behavior is somewhat different to when outside a form. That is, that they will automatically submit, whereas outside a form it will not.

You can add a type attribute to the button, as the example above <button type="button"></button> to stop it from submitting and just be a button, whereas the default for a button inside a form looks like this <button type="submit"></button>, even though we dont see those attributes.

There is another form of button that can be used inside a form though less common, is bying using the input element with the type set to either button or submit as demonstrated above.

The name attriubute

The name attribute is very important to a input element, as it sends an http request to the form element action attribute. You can think of name being a name for the input value. Something like a key:value pair. An example can be name="username" and the request will be username=whatever input was given from user.

Radio Buttons

  • <input type="radio" name="race" id="fun" value="half">
  • <label for="full">Full Race</label>
  • <input type="radio" name="race" id="full" value="full">

Radio button are like checkboxes except its either one or the other when it comes to selecting an option. The above illustration shows how this is done. You will notice the name attribute being the same, while the id's and values are different, this placing them under the same group.

The above illustration also takes the chance to demonstrate how the value attribute is being used. We see that although the name is the same, the values indicate the difference between each option. This because there is no user input other than clicking or selecting the option, thefore how do we differentiate between all options, hence adding a value attribute.
