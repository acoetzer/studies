Before talking aboutFunction Scopes, understanding that the wordscope refers to the location or visibility of a variable. In other words, it means where the variable is defined dictates where we have access to what variable.

Variables that are defined inside a function are scoped to that function. In Example 1, You see a variable scoped inside a function. This means, you do not have access to the variable outside the function. Unless its a global variable such as in Example 2.

When looking looking at Example 2, we have the same as before just with a global totalEggs variable defined. The outcome of this example would print out 6 followed by 0. This is because, even though the function was run first, the varaible scoped to that function had higher priority than the global variable.

This however, would not be the case if there was no variable scoped within the function, as in Example 3. This shows us that a function can call on its parent varaible, in this case the global variable.

This also brings us to another point and that is that a function can update a Global variable, as in Example 4. In other words, although variables inside a function are scoped to that function, They can read and update variables from outside its bubble.

Block Scopes refer to variables placed inside a block of code, such as inside the curly braces of a an if statement (which is known as conditionals), or a for loop.

Just like Function Scopes, You cannot refer the varaibles inside a block scope from the outside, you can vice versa. Though saying you cannot refer to or call a variable that is inside or scoped to a block of code, is kinda true depending on how you define your variable. Meaning if using var to define your variable it will be possible to refer to or call a varaible inside a block scope. However, this is not the case for function scopes when using var. If using let or const, you will not be able to refer to or call a variable scoped inside a block of code.
