Introducing Strings

  • let message = 'This is a string';
  • let message = "This is a string";
  • let message = 'this is a "quotation" in a string';
  • let message = 'This is a SyntaxError"
  • let message = '';

A string is a set of textual information, they represent text and must be wrapped in quotes and they are a another primitive type. You can put whatever text, symbol or number inside a string and it will be regarded as textual information, or in other words a string.

You can have either 'single quotes' or "double quotes" when creating a string, however you should remain consistant to which you use throughout your entire document. You cannot have 'starting and ending quotation set differently", this will result in a SyntaxError. Though you can have the opposite quotes inside the quotation marks, as shown above.

Another important thing to remember is that a number inside a string, is no longer a number, it is a string or considered as a string. You can also have a empty string as shown above.

Indices & length

  • let message = 'five';
    • message.length 4
    • message[0]
      • result is 'f'
  • 'string1' + 'string2'
    • 'string1string2'
    • Known as concatenation
  • 1 number type + 'hi' string type
    • '1hi' string type?

Strings within JavaScript have indices assigned to them, these indices begin from 0 upwards. Looking at the example board above, the string of five has 4 letter, however when indexing the text, it goes to a maximum of 3 as it starts from 0 and not 1.

To index a string, you would assign it to a variable and use square brackets next to it as so message[0].

The other thing a string has is what is known as a property, for example message.length whih will output the number of characters found within a string. Though this number begins from one and not zero completely different to indices.

Strings can also be added together by using the addition operator, this is known as concatenation. You can also add a number type to a string type, however JavaScript will try and find a common balance and sort of turn the number into a string.

String Methods

  • thing.method().method2().method3();
    • thing.toUpperCase();
    • thing.toLowerCase()';
    • thing.trim();

In JavaScript, Strings have actions known as methods. Each string can do the exact same as any other string. These methods can perform various things, such as making a string capitalized, uppercased, or search for something within a string . A string also has a property, now property and methods are different, one provides the property of the string while the other performs an action. The way you tell the difference between a method and property is the syntax, property does not end with parenthesis. The way you write a method is as so thing.method();.

You can chain methods together as seen in the above example board.

String Methods with Arguments

  • thing.indexOf();
  • thing.slice(beginIndex 5, [endIndex 10]);
    • The [ ]means optional
    • thing.slice(-6)
  • thing.replace('hi', 'bye')
  • thing.repeat(2)

When it comes to methods, there are some that take arguments(args) and some that dont, like the previous section above. Some of the examples above, are thing.indexOf('cat') which will provide the starting index of the first occurance for the pattern provided. Arguments can also provide a starting and end value, such as the thing.slice(5, 10), this stating that the slice of the string should happen the starting index of 5, and end at the ending index of 10 but not including. Also, it is important to remember that when reading the docs, [square brackets] mean optional.

Other examples of what you can do with arguments is, you can provide a pattern for replacing.. As an example, thing.replace('that', 'this'). Here you saying that you want to replace pattern that with pattern this.

The last example, is just providing a single argument and that is to repeat, thing.repeat(2). This will repeat whatever you have selected or chosen to repeat.

String Template Literals Super Useful

  • let product = 'milk';
  • let price = '$2';
  • let qty = 5;
  • without string Literals:
    • 'You bought' + ' ' + qty + 'of' + ' ' + product + ' ' + '. The total is :' + ' $' + qty * price
  • With String Literals:
    • ` You bought ${qty} of ${product}. The total is : $${qty * price}`

String Template Literals are ways for us to embed variables into our string without having to break the string up with concatenation. The most important to note is the use of ` back ticks `, located just above tab on the keyboard. In order to use string literals, you need to use back ticks, a normal string made with quotes will not work!.

Within the `back ticks` you use this ${literal} syntax to embed a variable.

Undefined & Null

  • Null
  • Undefined

Null and Undefined are part of the primitive types within JavaScript. Although both have this negative meaning behind it, that are vastly different from one another. Null is a intentional value that has to be set. Whereas Undefined is JavaScript's way of telling you I dont know. In otherwords Undefined is like variables that do not have an assigned value.

Math.random() gives us a random decimal number, something like 0.2989298289. Unlike other programming languages JavaScript does not have a random int, so to genereate a random interger requires some more work.

Looking at the example displayed above, we see that we can pass the Math object method as an argument inside a Math object method. After that we mulitply the random decimal to a whole number that we want to range between, and because the number will be floored, we add either a + 1 to include the final value, or we can specificy a particular range, as in from 21 to 23 in the example.
