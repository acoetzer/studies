Decision Making With Code

Decision making with code refers to things like, if a user is signed for example, then display this information. If one things equates to a certain value then something should happen. If there are comments, then display comments, or if there are no comments, then display something else instead.

So when talking about decision making in code, it refers to the a logical/conditional statement.

Comparison Operators

  • > Greater Than
  • < Less Than
  • >= Greater Than or Equal To
  • <= Less Than or Equal To
  • == Equality
  • != Not Equal To
  • === Strictly Equal To
  • !== Strictly Not Equal To
  • 5 > 3; true, it's greater than.
  • 5 < 3; false, it's not less than.
  • 5 >= 5; true, it's equal to.
  • 1 <= 5; true, it's less than.
  • 'a' < 'b' ; true, it's less than.
  • 'A' > 'a'; false, it's not greater than.

Comparison operators are like the math operators, though they do not perform any mathimatical function like addition. Comparison operators do however compare values. The first 4 comparison operators are fairly self explanatory, they check whether or not a value is greater than, less than or even less than or equal to and return a boolean value based on the result.

Another thing to mention is that, you can, but not recommended, compare 2 strings together. They will return a boolean value of true and false. This is due to Unicode, where each character as a value.

Equality: Triple Vs. Double Equals

The last for comparison operators have to do with quality, they check whether or not something has the same value or not. There are 2 sets, Doublt & Triple. The double equals == != Checks for equality of the value, though not for type. An example of this can be is the numerical 1 == string '1';, and the anwser is yes.

This is because of something called equality weirdness, JavaScript will coerce the values to a common type and usually it coerces to a string type, and therefor it will compare the Unicode values. Though, it becomes a little tricky, because if you had to say, is the numerical 1 == string '2';, then the answer is false and again we need to look at the Unicode value.

However, moving to triple equals, which is the preferred method for comparing if something is equal to or not. The triple, or otherwise known as strict equality, check for both value and type. It will not coerce to a common type. always use strict or triple equals!

If Statements

  • Conditionals
    • let userInput = prompt('What day is today?').toLowerCase();
      if (userInput === 'monday') {
      console.log("Oh No!, It's Monday!!!")
      } else if (userInput === 'friday') {
      console.log("Oh boy!!!, It's Friday!") } else {
      console.log(`Sorry today is ${userInput}. Still not Friday yet!!`)
      }

An if statement is a conditional, although here we just looking soley at the if statement. The basic gist of the condition is that code only runs when a conditional is true. When a condition is met, whatever has been specified to happen will happen. Taking a look at the example above, and soley at the if statement, or first block for if.

We are saying the if the variable userInput is equal to the string 'monday', then print using console.log a message. If that condintional is met, then it will perform the action of assigned to it, however, if it doesn't and there wasn't anything else, nothing will occur.

Else If

This brings us to the else if block. If we want more condintionals or outcomes for the same condition statement, then we can add an else if statement. Looking at the primitive example above, we see that if the condition for the first if statement was not met, then it can move to the next condintional an check whether or not it was met there. Thus, if met, will print and outcome, while if not met, will move over to the next conditional or else if or else.

Its important to note that you can have as many else if if you want.

Else

Once all else if statements has been used, and the userInput didn't match any of the desired results, you can place a else statement. An else state doesn not come with parenthesis and basically says, well fi your answer or result doesn't match anything we have above then sorry pal, move on.

  • falsy Value:

In Javascript everything has a truthyness or falsyness. There are only 2 values that are only either true and false, and that is true & false. Everything else is either truthy or falsey.

Logical AND

  • &&
  • true && true true
  • true && false false
  • false && false false
  • false && true false

    • The first Logical operater is AND represented with 2 ampersand &&. Both conditions have to be true inorder for the conditional statement to be true. JavaScript wil short circuit if the first conditional is not met. The same as when we nested the if statements.

    Going back to the password example where we nested our if statements we can use what is known as logical operators to achieve the same conditional. For example if (password.length >= 6 && password.indexOf(' ') === -1) {, This will provide the same outcome as when nested.

    Logical OR

  • true || true true
  • true || false true
  • false || false false
  • false || true true

    • Introducing logical OR which is respresented with ||. This is the complete opposite of &&, whereby both values need to be false inorder to be false. The statement is basically saying that one or the other condiction needs to be met in order to be true.

    Logical NOT
