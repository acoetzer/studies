The Position Property

  • position: static/relative/absolute/fixed/sticky;
    top: 50px;
    right: 10%;
    bottom: 4em;
    left: 0;

Above shows the property position and other properties that control the possitioning once a element has a position value set.

Static

  • position: Static;

An element with a value of static, will remain positioned to the normal flow of the document. The top, right, bottom and left properties will have no affect on this value, as this is the default value.

Relative

  • position: relative;

An element set with a relative position value, is positioned according to the normal flow of the document, however offset by the top, right, bottom and left properties relative to itself.

Absolute

  • position: absolute;

The element with the value of absolute is removed from the normal document flow, and no space is created for the element in the page layout. It is positioned to its closest already positioned relative. Otherwise it is relative to it's containing block, in other words the <body> element of the page. The final outcome or position is determined by the top, right, bottom and left properties.

Fixed

  • position: fixed;

An element with the position fixed, works similair to that of absolute. It is removed from the normal flow of the document, however is only relative to the containing block or <body> element. The one difference is the that it will remain in said position given by the top, right, bottom and left properties even when scrolling down and it will remain fixed to an element that also has a scrolling mechanism.

Sticky

  • position: sticky

A sticky element unlike fixed remains in the flow of the document and is offset in whatever postion it was given by the top, right, bottom and left properties. However sticky will stick to its nearest ancestor that has a scrolling mechanism.

CSS Transitions

  • transition: property/duration/transition-timing-function/delay;
    transition: color 0.25s ease-in 0.5s, border-color 0.25s ease-in 0.5s;
    • transition-property: color;
    • transition-timing-function: ease-in;
    • transition-delay: 0.5s;
    • transition-duration: 0.25s;

Transitions allow us to transition one property value to another property value. The transition property is a shorthand just like the border shorthand property. Above we can see the other properties associated with transition, you can single them out as well or use the shorthand. Good practise is to layout each property independantly as shown above with color and border-color, so that in the future if you change something it wont get in the way. You can always use the all value to apply the change to all property values.

The Power of CSS Transforms

  • transform: perspective(15px);
    transform: rotate(180deg), translate(1em);
    transform-origin: top left/50px 50px/bottom right 60px;
  • transform: rotate(0.5turn);
    • transform: rotateX(45deg)
    • transform: rotateY(180deg)
    • transform: rotateZ(360deg)
  • transform: scale(2, 0.5);
    • transform: scaleX(0.5)
    • transform: scaleY(1)
    • transform: scaleZ(2)
  • transform: skew(30deg, 25deg);
    • transform: skewX(5deg)
    • transform: skewY(50deg)
  • transform: translate(12px, 50%);
    • transform: translateX(12px)
    • transform: translateY(50%)
    • transform: translateZ(3em)

The transform property allows you to translate, rotate and scale elements in css. There is a transform shorthand along with multiple other associates that full under the shorthand. One being the tranform-origin property that allows you to set from an element should be scaled. As seen above. Transform also has multiple properties such as perspective, rotate, scale, translate and skew.

In short perspective, changes the perspective of the element. Rotate can rotate an element in 3d, or the x,y,z axis. Scale changes the size of an element. Translate moves the positioning of the element and skew skews the element.

The truth about background

  • background: no-repeat center/40% url(url to image or "../folder/image.png"), rose color background;
    • background-image: url(url to image)/linear-gradient(rgba(0, 0, 255, 0.5), rgba(255, 0, 175)), url(image);
    • background-size: auto/contain/scale/cover/30%/200px 100px;
    • background-repeat: no-repeat/repeat/repeat-x/space/round...;
    • background-position: bottom/top/left/right;

background is a shorthand for many other associated properties listed above. One thing to remember is that one of the properties (background-image) can set a gradients as well. That being said background can offer many various things, and one that we will look at is placing a background image. You place an image with using a value of url(url to image) or url("../local/file.png"). The shorthand accepts multiple properties like positioning and size, you can even add a background-color. Within the shorthand, order arrangement doesn't matter but it is important to remember when using background-size in the shorthand, it must be included after a potion separated with a forward slash / so an example would be center/cover orcenter/80%.
